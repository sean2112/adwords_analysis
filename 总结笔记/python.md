# Python 阶段adwords数据分析学习总结

## 一、阶段定位

在完整的数据分析流程（Excel → Python → MySQL → Power BI）中，Python 阶段承担的是：

- 数据清洗与标准化
- 数据结构重构（
- 数据质量控制自动化
- 数据建模准备

这一阶段的核心目标不是“分析结果”，而是：

> 构建一个可复现、可扩展、可验证的数据处理流程。

---

## 二、Python 阶段所学习到的核心知识点

### 1. ETL 思维的实践

ETL（Extract–Transform–Load）在本项目中得到完整实践。

#### （1）Extract（抽取）
- 使用 pandas 读取 Excel 数据
- 使用 pathlib 管理路径
- 验证读取是否成功（行列数、数据结构）

#### （2）Transform（转换）
- 文本标准化
- 类型转换
- 缺失值处理
- 异常值修正
- 百分比量纲统一
- 去重
- 构造代理主键
- 构造分类变量

#### （3）Load（加载）
- 输出 CSV 文件
- 统一编码格式
- 验证输出完整性

这一过程使数据处理从“手工操作”转化为“自动化流程”。

---

### 2. 数据类型管理能力

在 Python 阶段，重点学习了数据类型转换与控制：

- `to_numeric(errors="coerce")`
- `to_datetime(errors="coerce")`
- 使用 NaN 标记非法值
- 理解 NaN、None、NULL 的区别
  - None：Python 里啥都没有
  - Null：表格 / 数据库里缺数据
  - NaN：pandas里算错了，不是有效数字
  

这一能力的重要性体现在：

> 数据类型错误是后续模型与数据库错误的根源。

---

### 3. 缺失值与异常值的处理策略

学习到以下核心方法：

- 使用 `.isna()` 统计缺失值
- 使用 `clip()` 限制数值区间
- 将非法值统一转换为 NaN
- 在去重前定义业务主键

关键思维：

> 与其直接删除异常，不如先定义规则，再统一处理。

---

### 4. 文本标准化与数据一致性

通过字符串处理方法：

- 去除多余空格
- 合并连续空白字符
- 统一大小写
- 生成标准化字段（keyword_clean）

这一阶段强化了对“文本一致性”的理解：

> 机器判断一致性依赖规则，而非视觉直觉。

---

### 5. 量纲识别与统一

学习到通过数据分布判断比例字段的量纲：

- 判断是否为 0–1
- 或 0–100
- 再统一转换

这一能力属于：

> 数据语义理解能力，而非简单编程能力。

---

### 6. 数据建模准备能力

通过构造：

- 维表
- 事实表
- 代理主键

实现了从“单表数据”到“结构化建模数据”的转变。

关键理解：

> 数据建模始于数据清洗阶段，而非数据库阶段。

---

### 7. 数据质量门禁

使用断言（assert）验证：

- 外键是否存在缺失
- 主键是否唯一

这一机制体现了工程化思维：

> 让错误尽早暴露，而不是在下游系统中被动发现。

---

## 三、技术能力提升

通过 Python 阶段，掌握了以下技术能力：

### 1. pandas 核心操作能力

- 读取与写出数据
- 列选择与白名单控制
- 数据排序与重置索引
- 条件筛选与布尔索引
- 表连接（merge）
- 分组与聚合

### 2. numpy 基础应用能力

- 使用 NaN 表示缺失
- 向量化操作代替循环

### 3. 路径与文件管理能力

- 使用 pathlib 管理文件路径
- 避免硬编码路径问题

---

## 四、Python 阶段背后的数据分析思维

### 1. 可复现性优先

所有清洗规则必须写入代码，而不是依赖人工操作。

### 2. 明确规则优于模糊修正

每一个清洗动作都应有明确逻辑：

- 为什么要 clip？
- 为什么要转为 NaN？
- 为什么要去重？

### 3. 数据标准化是模型质量的前提

文本未标准化会导致：

- 重复统计
- 维表错误
- 关系建模失败

### 4. 先定义数据结构，再进入数据库

Python 阶段完成的是：

> 数据结构设计的第一步。

---

## 五、在未来项目中的经验教训

### 1. 不要跳过数据体检阶段

在任何数据项目中，都应先执行：

- 结构检查
- 类型检查
- 缺失检查
- 范围检查

### 2. 不要直接在原始数据上修改

所有清洗必须通过脚本完成，以保证可复现。

### 3. 永远显式转换类型

不要依赖自动识别。

### 4. 百分比字段必须统一量纲

这是常见错误来源。

### 5. 维表主键必须唯一

否则会影响数据库与 BI 阶段。

### 6. 不要过早优化可视化

Python 阶段的目标是结构化数据，而非展示结果。

---


## 六、阶段总结

Python 阶段的本质是：

> 将数据从“原始状态”转化为“可分析状态”。

这一阶段的质量决定了：

- 数据库结构是否稳定
- BI 模型是否可靠
- 分析结论是否可信

可以概括为：

> Python 阶段解决“数据是否可用”的问题。
